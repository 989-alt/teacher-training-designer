# 참고 서식: "지식샘터" 스타일 역량연계 교안 구조

지식샘터 컨설팅 강좌류(교사가 자신의 실천을 역량체계에 연계해 문서화하는 형식)를 여러 편 분석해 추출한 공통 구조. 최종 산출 문서는 이 구조를 뼈대로 삼되, 대상이 "학생 대상 단일 수업"이 아니라 "교사 대상 다차시 연수"이므로 ④⑤를 차시 수만큼 반복한다.

## 공통 구조

1. **표지**: [태그: OO 역량체계 기반] / 제목 / 부제(활용 도구) / 작성자·소속
2. **대상 및 목표**
   - 강의 목표 — 반드시 **인지적 / 정의적 / 행동적** 3분법으로 각 1개 이상 서술
   - 강의 대상 — 학년·직군·필요 역량 수준을 구체적으로 (예: "터미널 경험 없음, AI 챗봇 사용 경험 있음")
   - 활용 AI·디지털 기능 — 실제 도구명 나열
3. **역량체계 연계**
   - (1) 핵심 역량 최대 2개 선택 + 강조 표시
   - (2) 전체 세부지표 목록 중 실제로 다루는 것만 볼드로 강조 (한 역량의 지표를 전부 쓸 필요 없음 — `references/competency-framework.md`의 "관례" 참고)
   - (3) 역량 선택 기준 — 왜 이 조합인지 1~3문장으로 서술
   - (4) **[이 스킬에서 추가한 확장]** 지표 ↔ 활동 ↔ 관찰 가능한 행동 매핑표 — 원본 예시 교안에는 없지만, 심사·보고 목적상 "이 지표가 어느 활동에서 어떻게 드러나는가"를 명시적으로 요구받는 경우가 많으므로 항상 포함한다.
4. **맥락**
   - 기본 정보(대상 학년/교과 또는 대상 직군, 단원명 또는 연수 주제, 성취기준 또는 연수 목표)
   - 환경/실태(인프라 수준, 참가자 사전지식, 제약 조건)
   - 도구 선정 이유 — 왜 이 AI·디지털 기능을 선택했는지
5. **흐름 (차시별 반복)**
   - 도입 / 전개 / 정리 3단 카드, 각각 분 단위 시간 배분 명시
   - 전개는 Step1/Step2/Step3... 형태로 세부 활동을 순서대로 나열
   - 각 Step 옆에 관련 지표를 태그로 표기 (예: `[A1]`, `[G2 착수]`)
6. **차시별 실행 요약**: `[N차시] 소제목` 형태의 절 + 세부 흐름을 화살표로 이어진 단계로 표기

## 이 스킬이 원본과 다르게 확장하는 지점

- 원본(학생 대상 단일 수업 사례)은 1개 차시 또는 몇 개 차시의 "이미 실행한 수업"을 사후 문서화하는 데 초점. 이 스킬은 반대로 **아직 실행하지 않은 다차시 교원연수를 사전 설계**하는 데 초점을 맞춘다. 따라서:
  - 4번(맥락)의 "학급 실태" 항목은 "참가자 사전지식 수준 / 기기·환경 제약"으로 치환
  - 5번(흐름)을 차시 수만큼 반복하고, 차시 간 연속성(예: "2차시 착수 → 3차시 완성")을 명시
  - 운영 리스크 노트(설치·기기·시간 초과 등 현장에서 실제로 걸릴 만한 지점과 대안)를 마지막에 추가 — 원본에는 없지만 사전 설계 문서에는 필수적
